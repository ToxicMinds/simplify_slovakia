# Documentation

This folder contains Architecture Decision Records (ADR-style) and design artifacts.

## Index

- `vision.md` - Problem statement and product vision
- `scope-v1.md` - What's in/out for MVP
- `architecture.md` - System design and data flow
- `checklist-philosophy.md` - Why deterministic over AI-first
- `assumptions.md` - Known limitations
- `future-extensions.md` - Post-MVP roadmap
- `STEP_CONTRACT.md` - Step file format specification
- `design/` - Mind maps and diagrams

## Philosophy

> "If it's debated more than once, it belongs in docs/"

Documentation prevents re-litigation of decisions and enables new contributors to understand WHY, not just WHAT.
