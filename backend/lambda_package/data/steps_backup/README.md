# Steps

Atomic checklist items that represent a single bureaucratic action.

## Format

Each step follows the `STEP_CONTRACT.md` specification:

**Required fields:**
- `step_id` - unique identifier
- `title` - human-readable name
- `jurisdiction` - country/authority
- `applies_to.persona` - target user
- `preconditions` - what must be done first
- `outputs` - what this step produces
- `description` - what to do

**Optional fields:**
- `why_it_matters` - context/motivation
- `official_links` - government resources
- `failure_modes` - what can go wrong
